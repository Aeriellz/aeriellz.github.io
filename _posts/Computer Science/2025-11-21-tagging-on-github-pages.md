This was a problem from so long ago I actually am not mad about it anymore, which brings me to one more unexpected lesson: rage is temporary, knowledge is forever.  Why the rage, you ask? Because GitHub doesn't let you use Ruby plugins (nuance), and so tagging is a bit messy.

The nuance is that it kinda *does* let you? Maybe? You can use *some* plugins, but only if they're on the actual website for all the plugins, or something, it's not very clear — which brings me to my point: I am already bored and I'm not gonna figure that out. I have learned that there are solutions I will fight for, and solutions I just don't give a fuck about. I *could* go the extra mile and do the thing like that, but I don't need it, and I don't want it.

As someone who's spent most of my life doing things the hard way just because someone said it was the *right way* I call that progress. It's right if it works for me.

My solution was positively dumb: make a page for each tag, populate it. I used Liquid for the content, but procedurally generating a page for each tag right now is gonna be more work than creating them by hand. I can copy-paste as needed. I have fingers. It's fine. I'm running very few tags, and if I create any more I can just create a page for each of them. Five minutes to do that is not gonna ruin anyone's life, which is how some code bros make it feel like it will. 

Sometimes I feel like coders are allergic to *doing* things. I don't actually need or want the computer to do every single thing for me. I mean, Jesus, what's next? They're gonna create something dumb like a website that thinks for me and melts my brain— oh. Hold on, they *did* that?

Well, neverminding *that*, as stated, I actually *enjoy* doing things. It truly is kinda fun, and forces me to think very well about how and for what purposed I am tagging. A good structure has a tendency to solve most problems. It's also part of the learning, I've been realizing, that you need to find out how to scale your solutions to the size and specificities of your problem. In other words, you have a hammer, but not everything is a nail.

I've also played a bit with the styling and some CSS, adding some menus and it looked kinda cool, so I kept it for the "projects" section of each page. It still feels out of place, so I'll have to change a bit, but I was proud of that one. I actually had to write that code myself, and I never done anything like it. So I kept it more because of that. I had forgotten how it felt, to keep something just because you liked how it felt to do it, instead of how it looks like in the end. To give up perfection for the experience.

It's been slow going, getting this website in form, and making things I like, but I like it better. It's been fixing my relationship with time, slowly. Slow is the goal. You know, like a little plant. IT becomes what it becomes. Without pretence. It's not performative, I guess. It just will be what will be.

I kinda need more of that. So I'm pretty proud of what I've been doing here.
